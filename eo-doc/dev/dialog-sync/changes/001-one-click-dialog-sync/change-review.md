### [P2-1] 让无 target 调用边界可自动断言

- **类型**：测试边界补充
- **描述**：TODO-C1 当前主要覆盖按钮状态，TODO-S1 覆盖后端无 target 默认路径。实施时可把前端同步动作收敛到一个可注入或可测试的薄边界，使测试能够直接断言调用 `sync_provider_sessions` 时未传 `targetProvider`，减少只依赖 UI preview 手工验证的风险。

### [P2-2] 补充初始检查失败的视觉状态

- **类型**：场景补充
- **描述**：TODO-C3 已覆盖加载、可执行、执行中和已同步状态。实现视觉验证时可再覆盖初始 `provider_sync_snapshot` 失败，确认页面不会永久显示“检查中”，且错误消息与重试入口仍清晰可用。
